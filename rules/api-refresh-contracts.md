# 页面刷新与统计接口契约

## Purpose

- 防止页面操作、数据统计和接口调用边界混乱。
- 保证“刷新一个对象”只请求并更新这个对象需要的数据，“刷新当前页”只处理当前页，“刷新全部”才允许全量请求。
- 避免用前端全量请求再本地筛选来掩盖后端接口缺少过滤能力的问题。

## When to Read

- 修改管理端页面刷新、自动刷新、卡片/表格行刷新、统计汇总或趋势图。
- 新增或调整 `/usage/*`、`/auth-files/*`、`/models/*` 等管理 API 调用。
- 发现页面切换筛选、分页、tab 后数据没有按预期刷新。

## Rules

### 1) UI 动作必须先定义数据作用域

- 单卡片/单行按钮：作用域是当前实体，例如一个 auth file 的 `auth_index`、`source` 或 `name`。
- 当前页工具栏按钮：作用域是当前可见页项目集合。
- 当前筛选结果按钮：作用域是当前 filter/search 后的结果集合。
- 全量按钮：作用域才是全部数据，UI 文案必须明确是全量。

实现前必须能回答：这个动作会请求哪些接口、每个接口的过滤参数是什么、会更新哪些本地状态。

### 2) 后端过滤优先，禁止“全量拉取再前端筛”

- 如果 UI 作用域小于全量，API client 必须传明确过滤参数。
- 如果后端接口不支持过滤，先补后端过滤能力和测试，再接前端。
- 只有页面首屏初始化、显式全量刷新、导入导出这类全局场景可以使用全量接口。
- 统计接口尤其不能为了方便返回与当前实体无关的大量数据。

### 3) 查询参数应语义化而不是复用错误字段

- `api_key` 只能表示 API Key 维度过滤，不得拿来承载 auth file、provider、source 等其它概念。
- auth file 统计应优先使用稳定的 `auth_index`；兼容旧日志时可以补充 `source`。
- 多实体查询使用复数参数或重复参数，例如 `auth_index=a&auth_index=b`，不要拼接难解析的临时字符串。

### 4) 刷新必须区分触发原因

- 自动刷新可以跳过未过期缓存。
- 用户手动点击刷新必须强制请求目标作用域，不能被 stale 判断静默跳过。
- tab/filter/search/page 变化是可见数据上下文变化，不能只比较可见文件名；必要时把筛选条件和页码纳入刷新签名。
- filter/search 改变后通常应回到第 1 页，避免用户停留在旧分页导致误判“没有刷新”。

### 5) 状态合并只合并目标作用域

- scoped 请求返回后，只替换目标实体对应的本地统计，保留未请求实体的旧统计。
- 如果目标实体在返回中缺失，应明确把该目标统计置零或标记未知，不能继续展示过期值。
- 合并函数必须有单元测试覆盖“返回包含无关实体时不会污染其它卡片”和“返回缺失目标时不会保留旧值”。

### 6) 测试必须验证请求形状

- 不只验证 UI 最终显示，还要断言 API client 收到的路径和 query。
- 单实体刷新测试必须证明没有请求全量统计。
- 当前页刷新测试必须证明只传当前页实体集合。
- 筛选切换测试必须覆盖页码重置和可见页强制刷新。

### 7) 文件职责边界

- API client 只负责构造请求和规范化响应。
- 页面 hook 负责把 UI 作用域转换成请求作用域。
- 组件只负责触发语义化动作，不直接拼接口参数。
- 大组件或 helper 超过可读边界时，应优先抽出“作用域解析”“请求合并”“展示渲染”这类稳定单元。

## Auth Files 页面红线

- 单卡片 quota refresh 不得调用无过滤的 `/usage/entity-stats?days=30`。
- 当前页刷新不得因为缓存未过期而完全不请求用户刚要求刷新的卡片。
- 切换到“全部”必须让当前可见页进入新的刷新上下文；不能沿用上一筛选条件下的隐式缓存判断。
- `/usage/entity-stats` 如用于 auth files 页面，必须支持 `auth_index` / `source` 作用域过滤。
